# KNOWN_ISSUES — 알려진 제약과 폴백

1. **폰트 메트릭 차이로 인한 줄바꿈 미세 차이**
   원본 PPT가 사용한 Pretendard·맑은 고딕 등 미제공 폰트는 나눔스퀘어 네오로 매핑했다.
   글리프 폭 차이로 일부 문단의 줄바꿈 위치가 원본과 1~2자 다를 수 있다.
   콘텐츠 누락은 없으며 좌표·크기·색상은 원본 값을 그대로 사용한다.

2. **PowerPoint 전용 렌더링 효과의 근사 표현**
   - 3D 회전(scene3d/orthographicFront)은 평면 렌더링으로 대체 (해당 도형 2개)
   - 일부 커스텀 도형(custGeom 3건)은 SVG path로 변환 — 곡률이 원본과 미세하게 다를 수 있음
   - 패턴 채움(dkUpDiag)은 CSS repeating-linear-gradient로 근사
   - Wingdings 글머리 기호는 대응되는 유니코드 기호(●, ■, ➤ 등)로 치환

3. **로컬 정적 서버로 재생 시 영상 탐색(seek) 제한**
   Range 요청을 지원하지 않는 단순 정적 서버에서는 영상 중간 탐색이 느릴 수 있다.
   GitHub Pages와 file:// 직접 실행에서는 정상 동작한다.
   (QA 로그의 `media3.mp4 ERR_ABORTED`는 슬라이드 이탈 시 브라우저가 로드를 취소하는 정상 동작이다.)

4. **발표자 노트 창은 팝업 차단 시 열리지 않음**
   브라우저가 팝업을 차단하면 콘솔에 안내를 남기고 발표는 계속된다.
   해결: 주소창의 팝업 허용 후 N 키 재입력.

5. **대용량 미디어(총 ~330MB)로 초기 클론·로딩 시간**
   원본 pptx의 영상 13개(274MB)를 무손실 그대로 포함했다(재인코딩 도구 부재).
   video preload="none" 처리로 첫 화면 로딩에는 영향이 없으나, 저속 네트워크에서
   GitHub Pages 첫 방문 시 영상 슬라이드 재생 시작이 늦을 수 있다.
   오프라인 발표(로컬 실행)에서는 문제없다.

6. **GSAP는 로컬 번들로 포함되었으나 현재 미사용(예비)**
   전환·리빌은 CSS transform/opacity 애니메이션으로 구현되어 60fps를 만족한다.
   향후 Morph 등 고급 연출 확장을 위해 번들은 유지한다.
